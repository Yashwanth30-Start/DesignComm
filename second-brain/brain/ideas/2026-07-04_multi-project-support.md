# Idea: Multi-project support (importer + UI)

**Date:** 2026-07-04
**Status:** Deferred (Low)
**Source:** YAS-16

The data model already carries a `project` field per asset (Wing 1 / Wing 2), but the
UI can't yet filter/switch between projects at the top level. Low priority until a
**second real project's data actually exists** — building the switcher before there's
a second dataset would be structure-before-substance.
